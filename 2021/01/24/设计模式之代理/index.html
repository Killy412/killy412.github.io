设计模式之代理

发表于 2021-01-24 分类于 设计模式

概述

代理主要分为两种,静态代理和动态代理. 动态代理又分jdk动态代理和cglib动态代理.

静态代理是纯编码完成的,在编译完成的时候已经确认好类信息等.

动态代理是在运行时动态生成的类字节码,并且加载到jvm中.

静态代理

静态代理实现需要被代理类的接口,被代理类和代理类.

动态代理

jdk动态代理

jdk动态代理的关键类是 InvocationHandler 和 Proxy.

首先需要实现InvocationHandler接口来编写自己的业务逻辑.

invoke方法三个参数作用

  • Object proxy : 动态生成的代理对象
  • Method method : 需要代理的方法
  • Object[] args : 调用方法的参数
Proxy.newProxyInstance()方法是创建出一个代理对象.

三个参数作用

  • ClassLoader loader : 加载代理类的类加载器.
  • Class<>[] interfaces: 被代理类实现的接口
  • InvocationHandler h: 实现了InvocatinoHandler接口的对象

代码示例

被代理的类和接口依然用静态代理示例中的.

cglib动态代理

jdk动态代理只能代理实现了接口的类,而cglib(code generation library)代理没有这点限制. cglib使用的话需要引用第三方的jar包

cglib代理的核心类是MethodInterceptor和Enhancer.

参数作用

  • Object var1: 被代理的对象
  • Method var2: 被代理的方法
  • Object[] var3: 方法参数
  • MethodProxy var4: 用于调用原始方法

Enhancer 作用是创建出代理对象

代码示例

JDK代理和CGLIB代理比较

jdk动态代理只能代理有实现接口的类. 基于java反射机制实现. jdk层面支持.

cglib代理可以代理没有任何接口实现的类. 基于ASM机制实现,通过生成业务类的子类实现.
